Presseail: show latency ✎ ⓘ

This simple applet gives a way to visualize latency calculations in the rate-synchronous model. The idea is to (i) specify a chain element-by-element, (ii) to manually specify a schedule, and (iii) to see the resulting end-to-end latency values and generated constraints.

  1. Select a period for the chain input and click start.
  2. Add new links in the chain by specifying the operator, microcausality, and, if necessary, the sampling ratio.
    • direct: write-before-read within a period
    • last: read-before-write within a period
    • any: unconstrained sampling at same period
    • when: fast-to-slow rate transition (read after initial write)
    • when last: fast-to-slow rate transition (read before last write)
    • current: slow-to-fast rate transition
  3. Clicking on the slots sets the phase offset for the corresponding variable. The gray areas are excluded by scheduling and microcausality constraints. The pairwise latencies are shown above the reader arrows. The end-to-end latencies are shown at bottom, clicking on one highlights the corresponding path through the schedule.
  4. Add a latency bound and click update to generate ILP constraints on the end-to-end latency and causality, and to fix the schedule. While normally the constraints restrict possible schedules, here the idea is to validate the results. Click on solve to run glpk.js. Its log is shown at bottom. The left-hand-sides of certain inequalities are shown on the same line in gray. The values of the enabled chainlat constraints should correspond with the end-to-end latencies calculated by a direct algorithm and shown at the bottom of the main diagram.
  5. The simplify option applies simple constant propagation when generating constraints. The exists option generates a single set of constraints so that there need only exist a latency chain that satisfies the given bound.
  6. The small pencil button next to the title gives a way to copy-and-paste latency chains. Click the info button again to hide this text.
